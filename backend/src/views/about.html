About This Page

How the Application Works

The frontend sends HTTP requests to the Node.js/Express backend. The backend exposes REST endpoints, validates requests, applies business logic, and returns JSON responses.

Browser (Next.js UI) | | HTTP/JSON v Express API (routes/controllers/services) | | Mongoose queries v MongoDB

Technologies Used

Node.js, Express, JavaScript, Helmet, CORS, Morgan, and MongoDB (planned in next steps).

Main Limitations Right Now

This step only implements backend startup, health check, and this about page.

Alternative Approaches

We could have used Next.js API routes only. We chose Express to keep REST backend structure explicit and easier to defend in architecture discussions.
