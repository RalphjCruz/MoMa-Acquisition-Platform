About This Page

MoMAFlow

This page is served by the backend at /about. It documents the current assignment build, including architecture, workflows, roles, and implemented constraints.

Architecture Overview

Frontend (Next.js)

Catalogue, filters, auth, cart, buyer requests, manager tracking.

API (Node.js + Express)

REST routes for artworks, auth, users, and acquisitions.

Database (MongoDB)

Mongoose models for artworks, users, credentials, and acquisitions.

Role Workflows

  • Buyer: browse catalog, add to cart, submit pending purchase requests, view request status.
  • Manager (as admin): manage artworks (create/update/delete), review users, approve/acquire requests.
  • Catalog excludes items that are already in buyer cart, in requests, or already acquired.

Auth & Validation

  • JWT login/register with role-based route protection.
  • Registration requires username, email, role, and password.
  • Password rules: minimum 8 chars, at least one uppercase, at least one special character.
  • Register UI includes confirm-password validation.

REST Endpoints (Core)

/api/auth, /api/artworks, /api/users, /api/acquisitions

  • Artwork CRUD + search/sort/filter/pagination.
  • Acquisition status flow with manager-controlled approvals.
  • User listing/profile endpoints for manager view.

Assignment Alignment

  • Node.js/Express server with REST services.
  • React/Next.js client for testing and consuming all services.
  • MongoDB persistence using Mongoose models.
  • Full CRUD on artworks plus advanced features (search/filter/sort/pagination, role workflows, validation, responsive UI).
